# 文档变更分析 ## 1. 总体摘要 本次更新引入了新的 CLI 命令行参数、增强了 MCP 服务器的交互能力，并对 Hooks 系统进行了重大调整（主要是移除了交互式编辑功能），同时明确了不同模型（特别是 Sonnet 4.6 和 Opus 4.6）对自适应推理的支持。 ## 2. 关键主题 * **新增 CLI 启动参数**：支持通过 `-n` 在启动时命名会话，以及通过 `--effort` 设置单次会话的推理级别。 * **MCP 交互增强**：增加了 `Elicitation`（征询）相关文档，服务器现在可以在任务中途请求结构化输入（如表单或 URL 认证），且支持通过 Hooks 自动响应。 * **Hooks 系统调整**：`/hooks` 菜单改为只读浏览器模式，移除了通过菜单添加/删除 Hook 的功能，用户现在必须直接编辑 JSON 配置文件或让 AI 代劳。 * **模型能力扩展**：多处文档确认 Sonnet 4.6 现在与 Opus 4.6 一样支持自适应推理，并新增了仅限 Opus 4.6 的 `max` 努力级别。 * **配置与优化**：新增 `worktree` 相关设置以优化大型 Monorepo 的磁盘占用和启动速度，以及反馈调查频率控制。 ## 3. 影响级别 **Medium (中等)** **理由**：虽然核心功能流没有破坏性变更，但 Hooks 工作流的改变（从交互式菜单改为手动编辑 JSON）会影响用户的日常操作习惯。此外，新增的 MCP 征询功能和努力级别控制为开发者提供了更细粒度的控制手段。 ## 4. 开发者行动项 * **更新 Hooks 配置方式**：如果你习惯通过 `/hooks` 菜单管理 Hooks，现在需要改为直接编辑 `settings.json` 或让 Claude AI 帮你修改。 * **探索 MCP Elicitation**：如果你是 MCP 服务器开发者，可研究新的 `Elicitation` 功能以提供更好的中途交互体验；如果是使用者，注意新的弹窗交互流程。 * **利用新 CLI 参数**：在 CI/CD 或脚本中，使用 `claude -n <name>` 和 `--effort <level>` 来更精确地控制会话行为。 * **优化 Monorepo 工作流**：如果在大型 Monorepo 中使用 `--worktree`，考虑配置 `worktree.symlinkDirectories` 或 `sparsePaths` 以节省磁盘空间。
